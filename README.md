D66-64 Idle Zero + Auto Pedal Start

D66-63ベース。
漕いでいないのに時速が出る問題を修正。

原因:
- プロトタイプ初期値として st.rpm=58 / st.running=true が残っていた
- 手入力モードや初期表示で、実際には漕いでいなくても走行中扱いになっていた

修正:
- 起動時は必ず st.running=false / st.rpm=0 / jrPhysics.v=0
- センサー入力時、RPMが約5以上になったら自動走行開始
- RPMが0で惰性速度も0、さらに無入力が続いたら停止扱い
- センサー登録/自動再接続成功時は入力モードをセンサーRPMへ寄せる
- speed guardを追加し、RPMも惰性速度もない場合は必ず0km/h
- 通常状態の停止ボタン表示を「スタート」に修正

期待挙動:
- アプリを開いただけでは速度0
- 何も漕いでいなければ速度0
- 登録済みセンサーがあり、漕ぎ出してRPMを受信したら自動で走行開始
- 漕ぎやめ後は惰性減速
- 完全停止が続いたら停止待機

安全策:
- BLE解析本体は変更しない
- センサー登録UIは維持
- 標準走行モデル方針は維持
- Languageは維持
- 地図/追従/保存/読込/編集には触らない
- normalizeSavedRoute / loadMyRoutes / editMyRoute / useMyRoute / saveDraftRoute は触らない
- D66-48/D66-49の失敗CSSは含めない

検証済み:
- JS構文チェック
- 起動時idle強制処理存在
- 自動ペダルスタート処理存在
- 完全停止処理存在
- 速度0ガード存在
- センサー登録成功時のセンサー入力モード切替存在
- D66-63ユーザー向けセンサー設定維持
- D66-62標準走行モデル維持
- D66-60起動再帰修正維持
